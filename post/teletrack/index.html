Projet de stage

TeleTrack

Application web de gestion du télétravail

Projet réalisé dans le cadre d’un stage chez PORTNET S.A.

Java Spring Boot HTML5 CSS3 JavaScript Tailwind CSS H2 Database API REST

TeleTrack est une application web conçue pour simplifier la gestion des demandes de télétravail au sein de l’entreprise. Le projet permet aux employés de soumettre, consulter et suivre leurs demandes, tandis que les gestionnaires disposent d’un espace de pilotage pour approuver, rejeter ou laisser en attente les demandes, gérer les employés et visualiser les informations dans une interface claire et structurée.

Vue d’ensemble

Le projet répond à un besoin concret de digitalisation interne : centraliser la gestion des demandes de télétravail dans une plateforme unique, sécurisée et simple d’utilisation. L’application prend en charge l’authentification, la gestion des rôles, l’ajout et le suivi des demandes, ainsi que l’administration des employés.

Positionnement du projet : application web de gestion RH légère, avec logique métier claire, séparation frontend/backend et architecture orientée API REST.

Contexte entreprise

Ce projet a été réalisé dans le cadre d’un stage au sein de PORTNET S.A., entreprise spécialisée dans la digitalisation des procédures du commerce extérieur au Maroc. Le projet s’inscrit dans une logique de transformation numérique des processus internes, notamment autour de l’organisation du travail hybride et du télétravail.

Objectifs du projet

Objectifs fonctionnels

  • Faciliter la soumission et le suivi des demandes de télétravail
  • Permettre la validation ou le rejet des demandes par les managers
  • Centraliser les informations dans une interface ergonomique
  • Offrir une visualisation claire des statuts et plannings

Objectifs techniques

  • Mettre en place une architecture web moderne et claire
  • Assurer la sécurité via authentification et gestion des rôles
  • Garantir la cohérence des données dans la base H2
  • Préparer une base évolutive pour des améliorations futures

Fonctionnalités principales

Côté employé

  • Se connecter à l’application
  • Créer une demande de télétravail
  • Consulter ses demandes précédentes
  • Suivre le statut : Pending, Approved, Rejected

Côté manager

  • Consulter toutes les demandes
  • Valider, rejeter ou laisser en attente
  • Ajouter et gérer les employés
  • Superviser les demandes dans un dashboard
L’application inclut aussi l’importation des employés via CSV, une visualisation structurée des demandes et un fonctionnement orienté rôles pour séparer les droits d’accès.

Acteurs & rôles

Le système repose sur deux profils principaux.

Employé

  • Soumission des demandes
  • Consultation de l’historique
  • Suivi du statut de ses demandes

Manager / Webmaster

  • Validation ou rejet des demandes
  • Gestion des comptes employés
  • Suivi global des opérations

Architecture & stack technique

TeleTrack adopte une architecture web séparant clairement la partie interface, la logique métier et la couche de persistance.

Frontend : HTML5 CSS3 Tailwind CSS JavaScript Backend : Java Spring Boot API REST Spring Data JPA H2 Database Apache Tomcat IntelliJ IDEA VS Code

Frontend

HTML, CSS et JavaScript ont été utilisés pour construire une interface claire, responsive et simple d’utilisation, avec communication asynchrone via Fetch API / AJAX.

Backend

Spring Boot a permis d’implémenter la logique métier, la gestion des rôles, l’authentification et les endpoints REST reliés à la base H2 via JPA.

Workflow applicatif

Le fonctionnement de TeleTrack suit un cycle simple, compréhensible et directement lié à l’usage métier :

  • L’utilisateur s’authentifie selon son rôle
  • L’employé crée une demande de télétravail
  • La demande est enregistrée en base avec le statut Pending
  • Le manager consulte la demande dans son dashboard
  • Le manager approuve ou rejette la demande
  • Le statut mis à jour est renvoyé et visible côté employé
Ce workflow met en valeur une logique métier complète : saisie, persistance, supervision, décision et mise à jour en temps réel.

Consultation structurée des informations collaborateurs.

Persistance en base

Contrôle de la cohérence des données via H2 Console.
