Projet académique

Système de microfinance

Administration Oracle Database & interface Laravel

Oracle Database Oracle SQL Developer PL/SQL Laravel PHP MVC Sécurité Contrôle d’accès

Ce projet a pour objectif de concevoir et d’implémenter un système de gestion de microfinance sécurisé et cohérent, basé sur Oracle Database pour la couche données et Laravel pour la couche applicative. Le système couvre la gestion des clients, agents, prêts, échéances et remboursements, tout en intégrant des mécanismes de sécurité, de gestion des rôles, d’automatisation PL/SQL et de contrôle d’accès.

Vue d’ensemble

Le projet s’inscrit dans le cadre du module Administration des Bases de Données Oracle et applique les concepts étudiés à un cas réel : la gestion d’un système de microfinance.

L’enjeu principal est de construire une base de données robuste, fiable et sécurisée, tout en y connectant une interface web permettant d’interagir avec les données dans un cadre proche d’un système utilisé en entreprise.

Positionnement du projet : projet orienté administration de base de données, sécurité, PL/SQL et intégration applicative avec Laravel.

Domaine d’application

Le domaine de la microfinance a été choisi en raison de la complexité de ses données et de la sensibilité des informations manipulées. Le système doit gérer des entités liées aux clients, agents, prêts, échéances et remboursements, dans un contexte où la cohérence et la confidentialité sont essentielles.

Pourquoi la microfinance ?

  • Cas réel avec données interconnectées
  • Informations sensibles et critiques
  • Besoin fort de sécurité et de traçabilité
  • Bon terrain d’application pour Oracle et PL/SQL

Objectifs & besoins fonctionnels

Besoins fonctionnels

Pourquoi Oracle Database ?

Oracle Database a été choisi pour sa robustesse, sa fiabilité et sa forte présence dans les environnements professionnels. Le SGBD offre des mécanismes avancés pour la gestion des transactions, la cohérence des données, les accès concurrents et la sécurité.

Points forts d’Oracle

  • Gestion fine des transactions
  • Contrôle des utilisateurs, rôles et privilèges
  • Support natif de PL/SQL
  • Cadre réaliste pour l’administration de BDD

Environnement de travail

  • Oracle Database
  • Oracle SQL Developer
  • Laravel comme couche applicative
  • Interface web pour interagir avec la base

Utilisateurs, rôles & privilèges

Une partie essentielle du projet consiste à organiser les accès à la base de données à travers des utilisateurs distincts, des rôles et des privilèges. Cette approche permet d’appliquer le principe du moindre privilège et de protéger les informations sensibles.

Organisation logique

  • Création d’utilisateurs selon les profils métiers
  • Définition de rôles pour centraliser les permissions
  • Attribution contrôlée des privilèges système et objet
  • Utilisation de vues pour limiter l’accès direct aux tables

Exemples confirmés par le SQL

  • Rôles présents : ROLE_ADMIN, ROLE_AGENT
  • Tables métier : AGENT, CLIENT, ECHEANCE
  • Privilèges SELECT / INSERT / UPDATE / DELETE gérés selon les rôles
  • Contrôle d’accès appliqué au niveau base et application

Programmation PL/SQL

Le projet s’appuie sur PL/SQL pour centraliser la logique métier directement dans la base : procédures stockées, fonctions et triggers sont utilisés pour automatiser des traitements, garantir la cohérence des données et réduire les erreurs humaines.

Composants PL/SQL

  • Procédures stockées
  • Fonctions
  • Triggers
  • Gestion des exceptions

Rôle dans le système

  • Automatiser des contrôles métier
  • Mettre à jour certains statuts automatiquement
  • Encadrer la gestion des prêts et remboursements
  • Renforcer intégrité et fiabilité globales

Backend & Frontend Laravel

Laravel a été utilisé comme couche applicative afin de connecter l’interface web à Oracle Database, en s’appuyant sur une architecture MVC qui structure clairement le code entre modèles, vues et contrôleurs.

Structure des données

Une modélisation métier basée sur plusieurs tables centrales, notamment AGENT, CLIENT et ECHEANCE, avec des colonnes couvrant identité, coordonnées, dates, montants et statuts.

AGENT

Gestion des agents, identifiants, email, téléphone, login et mot de passe.

CLIENT

Informations client : identité, date de naissance, adresse, téléphone, email, type de client.

ECHEANCE

Suivi des échéances avec date, montant et statut, utilisé dans la gestion des prêts.
